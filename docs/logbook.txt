2026-06-16  -  SESSION 3: HOLOCALL FEATURE (v0.28 -> v0.30)
================================================================================
GOAL: call Jackie on a gig via a phone-call dialogue; he then spawns at a distance and
walks in. Asked whether to disable his "death flag" or mimic how other mods call NPCs.

NATIVE PHONE - SOLVED BY PROBING (full detail in docs/native_phone_probes.md):
- There is NO HolocallSystem class (Codeware Reflection: NOT FOUND). Calls run through the
  scriptable PhoneSystem: reachable via Game.GetScriptableSystemsContainer():Get("PhoneSystem").
- THE call entry point is:
    PhoneSystem:TriggerCall(questPhoneCallMode, Bool, CName callId, Bool, questPhoneCallPhase,
                            Bool, Bool, Bool, questPhoneCallVisuals)
  mode Video(2) = holocall. Phases: Undefined0 / IncomingCall1 / StartCall2(connected) / EndCall3.
  callId CName: "jackie" (alive) / "jackie_dead" (dead). visuals Default(0).
- In-game behaviour (captured live to probe_fires.txt via Codeware reflection dump + Observe):
  * IncomingCall on "jackie" -> game plays its OWN canned call (don't want).
  * StartCall -> a SILENT, persistent, see-through holocall window (our canvas). Stays till EndCall.
  * EndCall -> clean hang up.
  * Calling "jackie_dead" fires TriggerCall(Video, jackie_dead, IncomingCall) then EndCall ~13s
    later (rings, never connects) - so the dead contact is perfect to hijack (no canned content).
- KEY TECHNIQUE: probing was done by writing a Codeware reflection dump + Observe hooks INTO the
  mod (Config.probeNativePhone) that write to files in the deployed mod folder (phone_methods.txt,
  probe_fires.txt) which the dev reads directly. The CET console TRUNCATES big pasted blocks, so
  in-mod file output beats console copy/OCR.

CALL FLOW SHIPPED (Antonia's design, v0.30):
  RING (native IncomingCall ~2s) -> STOP (EndCall, aborts canned call) -> CONNECT (StartCall, the
  empty transparent window) -> our branching VOICE conversation runs over it -> end of any strand
  shows a RANDOM V farewell (12 text-only lines, Config.callFarewells) -> hang up (EndCall) ->
  "Let's do it." schedules the spawn/arrival. Watchdog + "Force hang up" button so a call can
  never get permanently stuck (the first attempt that kept StartCall open DURING the choice box
  got stuck - the connected window owns input; sequence avoids overlapping them).
- IMMERSIVE LINK: Observe PhoneSystem:TriggerCall; when the PLAYER calls Jackie from the in-game
  phone (IncomingCall on a 'jackie' id, not our own - reentrancy guard JL.call.selfTriggering),
